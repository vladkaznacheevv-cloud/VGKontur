# Следующий план действий

## Этап 1. Зафиксировать базовую стратегию

Нужно утвердить:

- рабочее название проекта;
- основной телефон/WhatsApp/Telegram;
- кто будет выезжать на замер;
- кто будет считать смету;
- список первых проверенных бригад;
- какие услуги точно берем в работу на старте;
- какие услуги пока только как дополнительные.

## Этап 2. Собрать точную рекламную статистику

В Яндекс Вордстат:

- проверить частотности по каждому городу;
- собрать похожие запросы;
- оценить сезонность;
- разделить mobile/desktop, если доступно.

В Яндекс Директ → Прогноз бюджета:

- задать регионы: Пятигорск, Кисловодск, Ессентуки, Железноводск;
- вставить стартовое ядро;
- добавить минус-слова;
- выгрузить XLS;
- оценить показы, клики, CPC, бюджет.

Google Keyword Planner:

- использовать только для справочной оценки ключевых слов и SEO, не как основной рекламный канал для РФ.

## Этап 3. Подготовить MVP сайта

MVP:

- лендинг;
- форма заявки;
- калькулятор;
- блок городов;
- блок услуг;
- блок преимуществ;
- блок строительного контроля;
- FAQ;
- политика конфиденциальности;
- дисклеймер по расчетам;
- интеграция заявки в Telegram;
- Яндекс Метрика;
- цели на отправку формы/клик по телефону/WhatsApp/Telegram.

## Этап 4. Подготовить рекламный тест

Стартовый тест:

- Яндекс Директ Поиск;
- бюджет 20 000–40 000 ₽;
- только горячие коммерческие запросы;
- отдельные объявления по городам;
- быстрый ответ на заявки;
- ручная квалификация клиентов.

## Этап 5. Подготовить операционные документы

Минимальный пакет:

- скрипт первичного звонка;
- чек-лист замера;
- шаблон сметы;
- шаблон договора;
- акт выполненных работ;
- чек-лист приемки этапов;
- чек-лист фотофиксации;
- таблица учета заявок;
- таблица учета объектов;
- регламент работы с бригадами.

## Этап 6. Первый цикл теста

30 дней:

- запустить рекламу;
- собрать заявки;
- посчитать стоимость лида;
- посчитать стоимость квалифицированного замера;
- провести первые замеры;
- получить первые договоры;
- сделать вывод: масштабировать, менять оффер, менять семантику или дорабатывать сайт.

## Главный принцип

Не строить большую компанию до проверки спроса. Сначала — дешево проверить спрос, отработать первые заявки, собрать реальные кейсы, затем постепенно усиливать сайт, рекламу и операционную систему.
